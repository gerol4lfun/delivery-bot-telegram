# Как перевести бота на Vercel (бесплатно)

Бот будет работать так же: раз в день пишешь ему даты — он обновляет Supabase. Но платить за сервер не нужно: Vercel запускает код только в момент, когда приходит твоё сообщение.

---

## Что нужно сделать по шагам

### Шаг 1. Зарегистрироваться на Vercel

1. Открой в браузере: **https://vercel.com**
2. Нажми **Sign Up** (Войти).
3. Выбери **Continue with GitHub** — войди через свой аккаунт GitHub.

---

### Шаг 2. Загрузить проект на GitHub (если ещё не загружен)

1. Зайди на **https://github.com**
2. Нажми **+** (правый верхний угол) → **New repository**
3. Название: например **delivery-bot-telegram**
4. Нажми **Create repository**
5. На странице репозитория нажми **uploading an existing file**
6. Перетащи сюда **все файлы** из папки с ботом (включая папки **api** и **lib**, файлы **package.json**, **parser.js**, **cities.js**, **supabase.js** и т.д.). Файл **.env** не загружай — он секретный.
7. Внизу нажми **Commit changes**

---

### Шаг 3. Подключить этот репозиторий к Vercel

1. Вернись на **https://vercel.com**
2. Нажми **Add New…** → **Project**
3. В списке найди свой репозиторий **delivery-bot-telegram** и нажми **Import**
4. Ничего не меняй в настройках (Root Directory оставь пустым, Build Command можно не трогать).
5. Нажми **Deploy**
6. Подожди 1–2 минуты. Когда появится галочка и надпись **Congratulations** — деплой готов. Нажми **Go to Dashboard** или открой свой проект.

---

### Шаг 4. Добавить переменные (секреты)

1. В проекте на Vercel открой вкладку **Settings** (слева).
2. Слева выбери **Environment Variables**.
3. Добавь по одной переменной (кнопка **Add** или **Add New**):

| Name | Value (подставь свои значения) |
|------|--------------------------------|
| `TELEGRAM_BOT_TOKEN` | Токен бота из @BotFather |
| `SUPABASE_URL` | Адрес твоего Supabase (например https://xxxxx.supabase.co) |
| `SUPABASE_SERVICE_ROLE_KEY` | Ключ service_role из Supabase |
| `ADMIN_USER_ID` | Твой Telegram user id (число). Можно узнать у @userinfobot в Telegram. |
| `WEBHOOK_SECRET` | Любое слово или фраза, которую только ты знаешь (например: мойСекрет123). Нужно только для шага 6. |

Для каждой переменной выбери окружения **Production** (и при желании Preview), нажми **Save**.

4. После того как все переменные добавлены: зайди во вкладку **Deployments**, нажми **⋯** у последнего деплоя → **Redeploy**. Так бот подхватит новые переменные.

---

### Шаг 5. Узнать адрес бота

1. Во вкладке **Deployments** открой последний успешный деплой (зелёная галочка).
2. Сверху будет ссылка вида: **https://delivery-bot-telegram-xxxxx.vercel.app** — это адрес твоего бота. Скопируй его.

---

### Шаг 6. Включить вебхук (один раз)

1. Открой в браузере новую вкладку и вставь адрес из шага 5.
2. В конец адреса допиши: **/api/setwebhook?secret=** и то слово, которое ты записал в **WEBHOOK_SECRET** (без пробелов).

Получится так:
```
https://delivery-bot-telegram-xxxxx.vercel.app/api/setwebhook?secret=мойСекрет123
```

3. Нажми Enter.
4. Должна появиться надпись вроде: **Вебхук включён: …/api/webhook**. Это значит, Telegram теперь шлёт сообщения на Vercel.

---

### Шаг 7. Проверить бота

1. Открой Telegram, найди своего бота.
2. Напиши **/start** — бот должен ответить.
3. Отправь тестовое сообщение с датами, например:
   ```
   Москва с 9.02
   Тула с 9.02
   ```
4. Если бот ответил и обновил данные — всё работает.

---

### Шаг 8. Отключить Railway (чтобы не платить)

1. Зайди на **https://railway.app**, открой свой проект с ботом.
2. Удали проект или останови сервис, чтобы с тебя больше не списывали деньги.

После включения вебхука (шаг 6) бот работает только через Vercel; Railway ему больше не нужен.

---

## Кратко

- **Vercel** — бесплатно, код запускается только когда ты пишешь боту.
- **Переменные** — те же, что были в Railway (токен бота, Supabase, ADMIN_USER_ID), плюс **WEBHOOK_SECRET** для страницы включения вебхука.
- **Один раз** открываешь в браузере ссылку **…/api/setwebhook?secret=…** — и бот переходит на работу через Vercel.

Если на каком-то шаге что-то непонятно — напиши, на каком шаге застрял и что видишь на экране.
